# Contributing

## Commit Message Convention

We use Conventional Commits for our commit messages. Just make sure your commits look like this:

```bash
type(scope): message
```

- `type`: The type of change (e.g., `feat`, `fix`, `docs`, `chore`, etc.)
- `scope`: Optional, a scope of the change (e.g., `web`, `api`, etc.)
- `message`: A short, concise description of the change.

### Some types of commit messages

- `feat`: A new feature
- `fix`: A bug fix
- `docs`: Documentation only changes
- `style`: Code style changes (e.g., formatting, missing semi-colons)
- `refactor`: Code changes that neither fix a bug nor add a feature
- `test`: Adding or updating tests
- `chore`: Routine changes (e.g., dependencies, build tasks)

Example Commit Messages:

- `feat(ui): add button component`
- `fix(api): resolve user login bug`
- `docs: update README with new setup instructions`

For more details on Conventional Commits, check out [https://www.conventionalcommits.org](https://www.conventionalcommits.org).
